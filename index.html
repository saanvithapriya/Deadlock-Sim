What is Deadlock?

A deadlock occurs when a set of processes are each waiting for a resource held by another process in the set, creating a circular wait.

4 Necessary Conditions

  • Mutual Exclusion — Only one process can use a resource at a time
  • Hold & Wait — A process holds resources while waiting for others
  • No Preemption — Resources can't be forcibly taken
  • Circular Wait — A cycle exists in the wait-for graph

DFS Cycle Detection

Uses Depth-First Search on the RAG. If a back edge is found (visiting a node already in the recursion stack), a cycle exists → deadlock.

Banker's Algorithm

For multi-instance resources: simulates resource allocation to find a safe sequence. If no safe sequence exists, the system is unsafe.

Welcome to DeadLock Sim

An interactive simulator for learning about Operating System deadlock detection & recovery.

1 Add Processes and Resources using the buttons in the panel.
2 Connect them using Add Link / Edge to create allocation and request relationships.
3 Run Detection to check for deadlocks, or use Step-Through to watch the algorithm step by step.
4 Try the Presets to see safe and deadlock scenarios instantly!
